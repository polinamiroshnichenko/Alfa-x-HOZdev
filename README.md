## Проблема
По итогам CustDev и анализа открытых источников, основной болью  представителей МСП является взаимодействие с юридическим и финансовым сектором. Особенно чётко эта проблема выражается на пересечении этих зон: в вопросах госзакупок и тендеров. 

### Актуальность проблемы
При увеличении количества МСП примерно на 10% за последние два года (период 2023-2025), менее 16 процентов участвуют в таких "сложных" программах;  почти четверть от общего числа заявок отвергается по формальным причинам; также конверсия количества заявок в победу в тендере или госзакупке составляет не более 22 %.

## Наше решение
Система персонализированного подбора подходящих программ, с возможностью задать все интересующие вопросы по документации nlp-помощнику

## Основные аспекты nlp-backend
1. Подбор релевантных тендеров для конкретного бизнеса - описание компании векторизуется, производится поиск по базе данных с эмбеддингами тендеров. 
2. Чат с документацией выбранного тендера - пользователь вводит запрос, он векторизуется, ищутся наиболее релевантные чанки текстов документации. Далее найденные фрагменты вместе с запросом пользователя добавляются в промпт и генерируется ответ.

### Какие базы данных используются в nlp части:
1. База данных с общей информацией о тендере - уникальный номер тендера, объект закупки, цена, даты и прочее. В ней также хранятся уже готовые эмбеддинги, по которым происходит поиск релевантных тендеров.
2. База данных с чанками - уникальный номер тендера, чанк текста документации этого тендера, эмбеддинг чанка.

### Какая embedding-модель используется: 
multilingual-e5-small 
+ быстрая, не требует мощной GPU
+ хорошо работает с русским языком
Возможные улучшения: заменить embedding-модель на SBERT, чтобы повысить качество работы с русском языком.

### Какая LLM используется:
Qwen3-14B 
+ не требует большого количества ресурсов (в отличие от, например, 30B)
+ точно следует инструкции, что важно для RAG
+ проще дообучить при необходимости (как это сделали, например, в Авито)

## Инструкция по запуску:
####Проект использует Git LFS для хранения больших файлов.  
Перед клонированием или работой с репозиторием установите Git LFS:
В консоли напишите:
git lfs install  

Если репозиторий уже клонирован без LFS — выполните:
git lfs pull

Далее клонируйте репозиторий:
git clone <repo_url>
cd <project_folder>

docker-compose up -d --build
### ВАЖНО! 
Проект использует shell-скрипт (init-db.sh) для инициализации базы данных внутри Docker-контейнера. Файловые системы Windows, Linux и macOS по-разному работают с правами файлов, и Git может их потерять.

Чтобы избежать ошибок при запуске, необходимо сообщить Git, что этот скрипт должен быть исполняемым.
Выполните эту команду в корне проекта:
git update-index --add --chmod=+x backend/init-db.sh

### Возможные проблемы и их решение:
Проблема 1: Ошибка permission denied или cannot execute: required file not found
Решение: локальное исправление 
chmod +x backend/init-db.sh

Проблема 2: Ошибка $'\\r': command not found или no such file or directory
Эта ошибка вызвана неправильными окончаниями строк (Windows-стиль CRLF вместо Unix-стиля LF).
Решение (через VS Code):
1. Откройте вашу папку backend в редакторе VS Code.
2. Найдите и откройте файл init-db.sh.
3. Посмотрите в правый нижний угол окна VS Code. Вы увидите надпись CRLF. 
4. Нажмите на эту надпись CRLF. Вверху экрана появится меню.
5. В этом меню выберите LF (Line Feed).
6. Надпись в правом нижнем углу изменится на LF.
7. Сохраните файл (нажмите Ctrl + S или Cmd + S).

Альтернативное решение (для Linux/macOS):
С помощью утилиты dos2unix:
Может потребоваться установка: sudo apt install dos2unix или brew install dos2unix
dos2unix backend/init-db.sh

## Основные пользовательские сценарии (MVP)

 1. Регистрация и авторизация
 + Новый пользователь вводит данные о своём бизнесе
 + Вводит логин и пароль, может войти в систему
 2. Подборка тендеров под бизнес
 + На главной странице выводится список тендеров, подходящих конкретно под тип бизнеса пользователя
 + Пользователь видит базовую информацию: регион, даты, цена

 3. Работа с карточкой тендера
 + Клик по карточке открывает страницу тендера
 + Нажатие на кнопку запускает ИИ-помощника, который генерирует краткое саммари
 + Пользователь может перейти на сайт, где размещён тендер

 4. Взаимодействие с ИИ-помощником
 + Пользователь задаёт уточняющие вопросы про тендер
 + ИИ отвечает на вопросы, разъясняет требования и риски

 5. Дополнительные возможности MVP
 + Настройки профиля бизнеса (редактирование данных)
 + Возможность выйти из аккаунта



## Roadmap на 3–5 месяцев

#### Расширение возможностей ИИ
 + Автоматический разбор тендерной документации (PDF/архивы)
 + Вложения в чате
 + Оценка рисков участия и вероятности победы
 + Создание  ИИ-агентов, которые помогают пройти полный цикл с тендером.
 + Подключение real-time парсинга тендерных площадок (государственные и коммерческие закупки).
 + Внедрение персонализированной рекомендательной системы - разработать алгоритм, который анализирует поведение пользователя и историю просмотренных тендеров, чтобы формировать индивидуальные рекомендации.
 
#### UX и функционал
 + Уведомления о новых тендерах
 + История и избранное
 + Расширенный поиск и фильтры по тендерам
 + Авто-проверка соответствия бизнес-профиля требованиям тендера
 +  Добавление модуля, отвечающего на вопросы, связанные с субсидиями, льготами, грантами и кредитными программами для малого бизнеса.
 + Добавление личного кабинета с историей выбранных тендеров, где можно будет отслеживать изменения в документации и датах.

#### Интеграции
 + Подключение веб- и Telegram-бота для доступа к тендерам
 + Возможная интеграция с банковскими сервисами (платежи, проверка финансовой устойчивости заказчика)

#### Монетизация
 + Тарифы по количеству доступных тендеров в день
 + Фиксированное количество токенов для работы ИИ-помощника
 + Процент от успешно заключённых сделок

#### Будущее расширение
 + Мобильное приложение после проверки устойчивости сервиса
 + Добавление универсальных ИИ-агентов для решения косвенных задач бизнеса
